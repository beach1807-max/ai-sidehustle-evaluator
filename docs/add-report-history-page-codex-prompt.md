# Codex 任務提示詞：新增「報告紀錄」頁面（一般 / 深度）

你現在要在既有的「副業點子冷靜評分器」專案中，新增一個可讓使用者回頭查看已產出報告的頁面。

## 一、任務目標

新增一個「報告紀錄」頁面，用來存放並查看使用者在本機瀏覽器中已產出的報告。

第一版不要做會員、不要做資料庫、不要做雲端同步、不要做登入。

請先使用 `localStorage` 儲存資料，讓使用者在同一台裝置、同一個瀏覽器中可以回頭查看之前產出的：

1. 一般報告
2. 深度報告

這一版目標是改善產品體驗，不是建立正式後台。

---

## 二、功能需求

### 1. 新增頁面

新增頁面：

```text
/reports
```

頁面名稱可顯示為：

```text
報告紀錄
```

導覽列或首頁入口需新增「報告紀錄」連結，讓使用者可以進入。

---

### 2. 報告儲存時機

當使用者成功產出報告後，自動將該報告存入 `localStorage`。

需要支援：

```text
一般報告 free
深度報告 deep
```

注意：
- 只有 API 成功回傳並成功渲染報告後才儲存
- API 失敗、JSON 驗證失敗、空資料不要儲存
- 範例報告是否儲存請不要預設儲存，除非目前專案已經有明確邏輯需要

---

### 3. 建議資料結構

請建立統一資料格式，例如：

```ts
type SavedReport = {
  id: string;
  type: 'free' | 'deep';
  title: string;
  idea: string;
  createdAt: string;
  summary?: string;
  score?: number | string;
  verdict?: string;
  report: unknown;
  input?: unknown;
};
```

其中：

- `id`：可使用 `crypto.randomUUID()`，若環境不支援則用 timestamp fallback
- `type`：一般報告為 `free`，深度報告為 `deep`
- `title`：優先使用副業點子名稱，沒有就顯示「未命名報告」
- `idea`：使用者原始輸入的副業點子
- `createdAt`：ISO string
- `summary`：用於列表摘要，從報告內容中抓取簡短文字，抓不到可留空
- `score`：若一般報告有分數則存入
- `verdict`：若一般報告有結論，例如做 / 縮小 / 放棄，則存入
- `report`：完整報告 JSON 或目前前端渲染所需資料
- `input`：使用者輸入內容，可選，但建議保存，方便未來做重新產出或複製

---

### 4. localStorage key

請使用固定 key：

```text
ai-sidehustle:saved-reports
```

如果專案已有 localStorage 命名規則，請遵守既有規則，但仍需避免與使用次數限制的 key 混在一起。

---

### 5. 儲存數量限制

避免 localStorage 過大，第一版最多保留最近 20 筆報告。

新增報告時：

1. 新報告放最前面
2. 超過 20 筆時刪除最舊資料

---

## 三、報告紀錄頁 UI 需求

### 1. 空狀態

如果沒有任何已儲存報告，顯示：

```text
目前還沒有報告紀錄
產出一般報告或深度報告後，會自動出現在這裡。
```

並提供返回輸入頁的按鈕：

```text
開始產出報告
```

---

### 2. 列表狀態

有資料時，以卡片列表顯示。

每張卡片至少包含：

```text
報告標題
報告類型：一般報告 / 深度報告
建立時間
簡短摘要 / 結論
查看報告按鈕
刪除按鈕
```

類型顯示建議：

```text
一般報告
深度報告
```

不要直接顯示英文 `free` / `deep` 給使用者。

---

### 3. 篩選功能

新增簡單篩選：

```text
全部
一般報告
深度報告
```

不需要做搜尋功能，避免第一版變複雜。

---

### 4. 查看單一報告

使用者點「查看報告」後，可以用其中一種方式處理：

優先方案：
- 導向既有報告頁，並用已儲存資料重新渲染

若既有架構不方便：
- 在 `/reports` 頁面內展開或顯示詳細內容

要求：
- 不要重新呼叫 API
- 不要重新消耗 Gemini API
- 必須使用 localStorage 裡已儲存的 report 資料渲染

---

### 5. 刪除功能

每筆報告提供「刪除」按鈕。

刪除前需有確認機制，例如 `window.confirm`：

```text
確定要刪除這份報告紀錄嗎？此操作無法復原。
```

刪除後更新列表。

---

### 6. 清空全部

頁面可加入「清空全部紀錄」按鈕，但要放在不容易誤按的位置。

按下前需二次確認：

```text
確定要清空所有報告紀錄嗎？此操作無法復原。
```

---

## 四、使用者提示文案

在報告產出成功頁面，若已成功存入紀錄，可顯示一行輕提示：

```text
已自動保存到報告紀錄，可稍後回來查看。
```

如果 localStorage 寫入失敗，不要中斷主流程，只需 console.warn，必要時顯示低干擾提示：

```text
報告已產出，但瀏覽器暫時無法保存紀錄。
```

---

## 五、技術要求

請依照現有專案架構實作，避免大改。

建議新增工具檔：

```text
src/lib/reportStorage.ts
```

或依照專案既有資料夾命名調整。

工具函式至少包含：

```ts
getSavedReports()
saveReport(report)
deleteSavedReport(id)
clearSavedReports()
```

需要處理：

- localStorage 不存在時不讓畫面崩潰
- JSON parse 失敗時 fallback 為空陣列
- 資料格式不正確時忽略錯誤資料
- 儲存失敗時不要影響報告產出流程

## 七、驗收標準

完成後請確認：

1. 一般報告成功產出後，會出現在 `/reports`
2. 深度報告成功產出後，會出現在 `/reports`
3. 重新整理頁面後，報告紀錄仍存在
4. 點擊查看報告不會重新呼叫 API
5. 刪除單筆報告後，該筆資料消失
6. 清空全部後，列表變成空狀態
7. 最多只保留最近 20 筆
8. `npm run build` 通過
9. 不影響既有一般報告、深度報告、使用次數限制、回饋表單功能
