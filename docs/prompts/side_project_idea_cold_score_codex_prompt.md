# 副業點子冷靜評分器｜Codex Mock 版開發提示詞

你是一位資深前端工程師與產品設計師，請幫我建立一個純網頁 Mock 版產品。

---

## 產品名稱

副業點子冷靜評分器

---

## 產品目標

讓使用者輸入一個副業點子後，得到一份結構化、冷靜、偏殘酷但有建設性的副業可行性評估報告。

這個產品不是「創業靈感產生器」，而是「副業點子風險健檢工具」。

核心價值是：

> 幫使用者在投入時間之前，判斷這個副業點子該做、該縮小，還是該放棄。

---

## 第一版限制

請先做 Mock 版，不要接任何真正 AI API。

第一版請遵守以下限制：

- 純前端網頁
- 不做登入
- 不做會員
- 不做付款
- 不做資料庫
- 不做後台
- 不接 OpenAI API
- 不需要真正儲存使用者資料
- 使用 mock data 呈現報告效果
- 使用者送出表單後，導向報告頁並顯示固定範例報告

## 整體設計風格

風格要乾淨、冷靜、專業、有顧問感。

不要做得太可愛，也不要像一般 AI 聊天工具。

視覺關鍵字：

- 冷靜
- 理性
- 殘酷但有幫助
- 顧問報告
- 一人副業者適用
- 低風險 MVP
- 避免浪費時間

可以使用卡片式設計、分數區塊、標籤、進度條、風險提示框。

整體色系建議：

- 背景：淺灰或白色
- 主色：深藍、黑灰、冷色系
- 風險提示：可使用紅色或橘色
- 成功 / 可做提示：可使用綠色
- 中性提示：灰色

# 報告頁需求

路由：`/report`

報告頁要像一份顧問報告，不要像聊天回答。

請包含以下區塊：

---

## 1. 使用者輸入摘要

```text
你正在評估的點子

```

## 2. 冷靜總分

用大型分數卡呈現：

```text
74 / 100
```

旁邊顯示結論標籤：

```text
可做，但必須縮小範圍
```

請加上簡短解讀：

```text
這個點子有真實需求與付費可能，但不能做成完整推薦平台。第一版應該縮小成「飼料成分分析報告產生器」。
```

---

## 3. 七大維度評分

請用卡片或進度條呈現：

- 個人契合度：13 / 15
- MVP 難度：15 / 20
- 需求強度：16 / 20
- 付費可能性：11 / 15
- 獲客難度：10 / 15
- 法律與平台風險：5 / 10
- 維護成本：4 / 5

每個維度都要有一句簡短說明。

---

## 4. 一句話結論

內容：

```text
可以做，但不要做成「寵物營養推薦平台」。第一版應該先做成「飼料成分分析報告產生器」，驗證是否有人願意為單次報告付費。
```

---

## 5. 最大優勢

內容：

```text
你自己就是目標客群，且寵物飼料選擇確實會帶來焦慮。這類問題有明確情境，例如軟便、過敏、挑食、體重控制，容易包裝成單次分析報告。
```

---

## 6. 最大風險

內容：

```text
這個題目容易碰到寵物健康、營養與醫療責任。產品不能宣稱治療、診斷或保證改善，只能定位為教育型輔助分析工具，並提醒使用者必要時諮詢獸醫。
```

---

## 7. 致命紅旗

請用警示卡片呈現：

- 不要做成醫療診斷工具
- 不要承諾改善軟便、過敏或疾病
- 不要一開始建立完整飼料資料庫
- 不要做品牌排行榜，避免爭議與維護成本
- 不要用「推薦唯一最佳飼料」這種高責任說法

---

## 8. 建議縮小後的 MVP

內容：

第一版只做「飼料成分分析報告」。

使用者輸入：

- 狗狗年齡

- 目前狀況，例如軟便、挑食、過敏疑慮、體重控制

系統輸出：

- 成分摘要
- 可能優點
- 可能疑慮
- 適合度評估
- 需要注意的地方
- 非醫療免責聲明
