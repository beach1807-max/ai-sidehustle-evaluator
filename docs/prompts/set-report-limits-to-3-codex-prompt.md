# Codex 任務提示詞：將一般報告與深度報告使用次數都調整為 3 次

你現在要在既有的「副業點子冷靜評分器」專案中，調整使用次數限制。

## 一、任務目標

將目前的一般報告與深度報告使用次數限制，統一調整為：

```text
一般報告：3 次
深度報告：3 次
```

這次只做使用次數限制調整，不要新增會員、金流、資料庫或其他功能。

---

## 二、需要檢查的範圍

請檢查目前專案中所有跟使用次數限制有關的地方，包含但不限於：

```text
localStorage usage limit key
一般報告 free report limit
深度報告 deep report limit
按鈕 disabled 條件
剩餘次數顯示文案
錯誤提示文案
回饋解鎖深度報告邏輯
```

如果目前一般報告是 2 次、深度報告是 1 次，請改成兩者都是 3 次。

---

## 三、邏輯要求

### 1. 一般報告

一般報告最多可成功產出：

```text
3 次
```

成功產出後才扣次數。

API 失敗、timeout、503、429、JSON parse 失敗，不應扣次數。

---

### 2. 深度報告

深度報告最多可成功產出：

```text
3 次
```

成功產出後才扣次數。

API 失敗、timeout、503、429、JSON parse 失敗，不應扣次數。

---

### 3. 顯示文案

請同步更新前端顯示文案，例如：

```text
今日一般報告剩餘 3 次
今日深度報告剩餘 3 次
```

或依照目前專案既有文案風格調整。

不要出現舊文案，例如：

```text
一般報告 2 次
深度報告 1 次
```

---

### 4. localStorage 舊資料處理

如果使用者瀏覽器已經有舊的 localStorage 使用紀錄，請確認不會造成畫面錯誤。

這次可以不強制清除舊紀錄，但請確保新的上限為 3。

例如：

```text
已使用一般報告 2 次 → 新上限 3 → 應剩餘 1 次
已使用深度報告 1 次 → 新上限 3 → 應剩餘 2 次
```

---

## 四、不要做的事情

這次不要做：

```text
會員登入
資料庫
雲端同步
付款功能
報告紀錄頁
AI API retry 改造
UI 大改版
```

只調整一般報告與深度報告的使用次數限制。

## 五、驗收標準

完成後請確認：

1. 一般報告最多可成功產出 3 次
2. 深度報告最多可成功產出 3 次
3. 一般報告與深度報告的剩餘次數顯示正確
4. 成功才扣次數
5. API 失敗、timeout、503、429、JSON parse 失敗不扣次數
6. 舊的 2 次 / 1 次文案已全部移除
7. `npm run build` 通過
8. 不影響回饋表單、報告紀錄頁、一般報告、深度報告既有流程
