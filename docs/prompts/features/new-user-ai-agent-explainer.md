# 新手導引用語優化：AI Agent 基本介紹與白話說明

## 任務目標

目前產品已經可以產生：

* 免費副業點子評估
* Deep Report
* AI Agent 開工包
* 可複製給 AI Agent 的開發任務

但對完全新手來說，可能不理解：

* AI Agent 是什麼
* MVP 是什麼
* Prompt 是什麼
* AI Agent 開工包可以拿來做什麼
* 為什麼這份內容有價值

本次任務目標是：

讓完全沒有開發經驗的使用者，也能理解：

```text
這不是單純分析報告。

這是一份可以交給 AI 工具，協助產出第一版 MVP 網站的開發說明書。
```

---

## 修改原則

### 不要做

本次不要新增：

* 金流
* 會員
* 登入
* 資料庫
* Email
* 使用次數限制
* 新 API
* 新 AI 串接
* 新報告欄位

### 要做

只做：

* 文案優化
* 新手說明
* 名詞白話化
* 使用流程說明
* CTA 文字調整

---

## 一、首頁加入 AI Agent 基本介紹

如果目前首頁或 evaluate 頁面有產品介紹區塊，請加入一個簡短區塊。

### 標題

```text
什麼是 AI Agent？
```

### 內容

```text
AI Agent 可以理解成「會幫你執行任務的 AI 助手」。

一般 AI 聊天是你問一句、它回一句。

AI Agent 更像是：
你給它一份清楚的任務說明，它可以幫你建立檔案、修改程式、整理架構，甚至一步步做出第一版網站。
```

### 補充說明

```text
本工具會幫你把副業點子整理成 AI Agent 看得懂的開發任務，讓你不用從零開始寫需求。
```

---

## 二、Deep Report 頁面加入「這份報告怎麼用？」

在 Deep Report 最上方加入新手導引區塊。

### 標題

```text
這份報告怎麼用？
```

### 內容

```text
如果你是新手，不需要一次看懂所有內容。

建議照這個順序：

1. 先看「是否值得做」，確認這個點子不要太大、太貴、太難維護。
2. 再看「MVP 功能清單」，知道第一版只需要做哪些功能。
3. 點擊「AI Agent 開工包」。
4. 複製完整的開發任務說明。
5. 貼到你使用的 AI 開發工具。
6. 讓 AI 先做出第一版網站框架。
```

---

## 三、AI Agent 開工包頁面加入說明

在 AI Agent 開工包頁面，於「複製全部」按鈕上方加入說明。

### 標題

```text
這是什麼？
```

### 內容

```text
這是一份整理好的 AI 開發任務說明。

你可以把它整段複製，貼到 Codex、Cursor、Claude Code、ChatGPT Agent 或其他 AI 開發工具。

AI 會根據這份說明，協助你建立第一版 MVP 網站框架。
```

### 重要提醒

```text
第一版不會幫你完成正式金流、會員、資料庫或外部 API 串接。

它會先建立可操作的框架、假資料流程、設定範例與 README，讓你可以先看到產品長什麼樣子。
```

---

## 四、新增「新手名詞解釋」區塊

請在 Deep Report 或 AI Agent 開工包頁面加入一個小卡片。

### 標題

```text
新手名詞解釋
```

### 內容

```text
AI Agent
= 會幫你執行任務的 AI 工具

Prompt（提示詞）
= 你交給 AI 的任務說明

MVP（最小可行產品）
= 先做出最簡單、可以測試市場反應的第一版

AI Agent 開工包
= 幫 AI 準備好的施工說明書，包含功能、頁面、開發步驟與驗收條件

Mock（假資料 / 假流程）
= 先用假的資料或假的流程做出畫面，之後再接真的服務
```


6. MVP 有白話解釋。
7. Mock 有白話解釋。
8. 開工包按鈕與文案比原本更直覺。
9. 不影響原本 Deep Report 產生流程。
10. 不影響 AI Agent 開工包複製功能。
11. Build 通過。
