# Codex 任務：階段 1－新增付費深度報告預覽，不接金流

## 任務目標

請在目前的副業點子評估工具中，新增「付費深度報告預覽」區塊。

本階段只做前端內容架構，不接金流、不新增資料庫、不新增會員、不修改 AI API 行為。

目標是先測試使用者是否對「副業點子深度驗證報告」有興趣。

---

## 目前產品定位

免費版維持現在流程：

```text
輸入副業點子
→ 得到冷靜評分
→ 告訴使用者該做 / 縮小 / 放棄
```

免費版是引流工具，不要縮減現有免費結果內容，也不要破壞目前可用功能。

---

## 本次要新增的內容

請在免費分析結果下方，新增一個「付費深度報告預覽」區塊。

建議標題：

```text
想把這個點子變成可執行 MVP？
```

副標：

```text
解鎖副業點子深度驗證報告，取得更具體的 7 天執行方案與 Codex 開發提示詞。
```

內容清單：

```text
解鎖深度報告後，你會得到：
1. 7 天 MVP 行動計畫
2. Codex 開發提示詞
3. 一頁式銷售頁文案
4. 第一版收費建議
5. 風險修正方案
```

價格文案先顯示：

```text
單次深度報告：NT$49～99
```

按鈕文字：

```text
我想要這份深度報告
```

按鈕點擊後，不要導向付款頁，不要呼叫 API，只顯示提示訊息：

```text
此功能即將開放，感謝你的興趣。
```

提示訊息可以用 toast、alert、modal、inline message，依照目前專案已有的 UI 寫法選擇最小改動方案。

---

## 視覺與 UX 要求

1. 付費區塊要明顯區隔於免費報告，但不要太像強迫推銷。
2. 可以使用卡片、邊框、背景色或徽章標示「深度報告預覽」。
3. 手機版與桌機版都要可讀。
4. 不要影響目前免費分析結果的閱讀順序。
5. 不要要求使用者登入。
6. 不要新增付款欄位。
7. 不要新增 Email 欄位。
8. 不要新增等待名單功能。

---

## 技術限制

本階段不要做以下事情：

```text
不要接 Stripe
不要接綠界
不要接 Line Pay
不要新增資料庫
不要新增會員系統
不要新增使用次數限制
不要修改 Gemini Prompt
不要新增 deep report API
不要改變現有 evaluate API 輸入輸出格式
```

請只做必要的前端變更。

---

## 建議實作方式

請先檢查目前專案結構，找出分析結果顯示元件。

可能位置包含但不限於：

```text
src/App.tsx
src/components/*
src/pages/*
```

請優先採用最小改動：

1. 如果已有結果顯示元件，直接在結果區塊下方新增 PaidReportPreview 元件。
2. 如果目前沒有拆元件，可以新增一個小型元件：

```text
src/components/PaidReportPreview.tsx
```

3. 樣式請沿用現有專案風格。
4. 若專案使用 Tailwind，就用 Tailwind。
5. 若專案使用 CSS module 或一般 CSS，就沿用現有方式。

---

## 互動行為

按下「我想要這份深度報告」後：

- 不送出資料
- 不呼叫後端
- 不跳轉頁面
- 只顯示「此功能即將開放，感謝你的興趣。」

若使用 React state，可用：

```text
showComingSoonMessage
```

顯示在按鈕下方即可。

---

## 驗收標準

完成後請確認：

1. 免費版評估流程仍可正常使用。
2. 分析結果下方有出現付費深度報告預覽。
3. 按鈕點擊後會顯示「此功能即將開放，感謝你的興趣。」
4. 沒有新增付款流程。
5. 沒有新增資料庫。
6. 沒有修改 Gemini API 或 evaluate API。
7. `npm run build` 可以通過。
8. 若有 lint/test 指令，也請執行並回報結果。
