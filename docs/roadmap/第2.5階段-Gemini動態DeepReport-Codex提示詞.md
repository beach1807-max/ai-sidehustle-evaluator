# 第 2.5 階段：Deep Report 串接 Gemini 動態內容

## 任務目標

將目前 Deep Report 從固定 Mock Data 升級為 Gemini 動態產生。

目前畫面與結構維持不變。

本階段重點是讓 Deep Report 內容與使用者輸入的副業點子高度相關。

---

## 目前狀態

已完成：

- free mode
- deep mode
- Deep Report UI
- Deep Report JSON Schema
- Deep Report Preview 頁面

目前仍使用：

```text
AI_PROVIDER=mock
```

導致所有 Deep Report 幾乎相同。

---

## 本階段目標

改為：

```text
AI_PROVIDER=gemini
```

時：

Deep Report 內容由 Gemini 產生。

---

## 保留項目

禁止修改：

- 免費版 Prompt
- 免費版報告格式
- Deep Report UI
- 路由結構
- JSON Schema

---

## 新增 Deep Report Gemini Prompt

建立專用 Prompt。

角色設定：

```text
你是一位冷靜、務實、偏保守的產品策略顧問。

你的任務不是鼓勵創業。

而是協助使用者：

- 判斷是否值得做
- 如何縮小 MVP
- 如何降低風險
- 如何用 AI Agent 快速完成第一版產品
```

---

## Deep Report 必須根據點子產出

以下內容禁止固定模板。

必須依據輸入內容產生。

### MVP 可行性分析

必須依據：

- 點子類型
- 客群
- 開發難度
- 維護成本

調整結果。

---

### MVP 功能清單

不同點子產出不同功能。

例如：

寵物飼料分析

```text
搜尋
評分
比較
```

AI 漫畫平台

```text
作品頁
角色頁
生成頁
```

---

### 7 天行動計畫

必須根據產品類型調整。

禁止固定 Day1~Day7。

---

### AI Agent MVP 啟動包

必須根據點子內容產出：

- 產品目標
- 客群
- MVP 功能
- 頁面需求
- UI需求
- 資料結構
- 技術限制
- 驗收條件

---

### 首頁銷售文案

必須根據產品調整：

- 主標題
- 副標題
- 功能特色
- CTA

---

### 收費建議

根據產品類型調整。

例如：

工具型產品：

```text
單次付費
訂閱制
```

內容型產品：

```text
一次性購買
```

---

### 獲客建議

必須根據客群調整。

例如：

寵物產品：

```text
Facebook 狗社團
PTT Dog
寵物論壇
```

開發工具：

```text
Threads
Indie Hackers
Reddit
```

---

## 新增 AI Agent 執行策略

新增區塊：

```json
{
  "agentExecutionStrategy": {}
}
```

內容包含：

- 推薦技術
- 推薦部署平台
- 建議開發順序
- 預估頁面數
- 預估檔案數
- MVP 完成條件

範例：

```text
技術：React + Vite

部署：Cloudflare Pages

順序：
首頁
輸入頁
結果頁

頁面數：4

檔案數：10~15
```

---

## JSON Schema 更新

新增：

```json
{
  "agentExecutionStrategy": {}
}
```

保持向下相容。

---
